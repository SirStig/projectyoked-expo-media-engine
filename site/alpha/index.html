Feature overview

  • Real-time preview — MediaEnginePreview renders at ~30 fps using the same pipeline as export. Filters, transitions, speed, and opacity all match.
  • Multi-track composition — Stack video, audio, image, and text tracks with per-clip timing, blending, and transforms.
  • 9 filters — Grayscale, sepia, vignette, invert, brightness, contrast, saturation, warm, cool. All with adjustable intensity.
  • 8 transitions — Crossfade, fade to black, slide (left/right/up/down), zoom in/out. Configurable duration.
  • Per-clip controls — Opacity, speed, source trimming, position, scale, rotation, resize mode.
  • Audio mixing — Multiple tracks with volume, fade in/out, and keyframe-based automation.
  • Text & emoji overlays — Font size, bold, color, shadow, stroke, background, timed visibility.
  • Image overlays — Positioned, scaled, rotated image tracks with timing.
  • Video stitching — Fast concatenation with transcoding fallback.
  • Video compression — H.264 or H.265, quality presets or explicit bitrate.
  • Waveform generation — Normalized RMS amplitude array from audio.
  • Audio extraction — Pull audio from video to .m4a.

Preview engine

The preview system is designed for a CapCut-style layered editor:

  1. <MediaEnginePreview> renders video tracks natively — filters, transitions, and speed are applied in hardware, matching the export output exactly.
  2. useCompositionOverlays(config, currentTime) returns the active text and image clips with all transforms resolved, so your Skia or Reanimated layer can render them interactively on top.
  3. The same CompositionConfig drives both layers and the final export — no coordinate conversion, no guessing.
import { useRef, useState } from 'react';
import { View, StyleSheet } from 'react-native';
import {
  MediaEnginePreview,
  useCompositionOverlays,
} from '@projectyoked/expo-media-engine';

export function CompositionEditor({ config }) {
  const previewRef = useRef(null);
  const [currentTime, setCurrentTime] = useState(0);
  const [isPlaying, setIsPlaying]     = useState(false);

  // Active text and image clips at the current position,
  // with x/y/scale/rotation/opacity resolved — feed to Skia directly
  const overlays = useCompositionOverlays(config, currentTime);

  return (
    <View style={styles.container}>
      {/* Native video layer — filters, transitions, speed applied here */}
      <MediaEnginePreview
        ref={previewRef}
        config={config}
        isPlaying={isPlaying}
        onTimeUpdate={e => setCurrentTime(e.nativeEvent.currentTime)}
        onLoad={e => console.log('Duration:', e.nativeEvent.duration)}
        style={StyleSheet.absoluteFill}
      />

      {/* Interactive overlay layer — render with Skia, Reanimated, or RN */}
      {overlays.map(overlay => (
        <InteractiveOverlay key={overlay.id} overlay={overlay} />
      ))}
    </View>
  );
}

composeCompositeVideo

The primary export function. Renders a CompositionConfig to a video file.

import MediaEngine from '@projectyoked/expo-media-engine';

const outputUri = await MediaEngine.composeCompositeVideo({
  outputUri: 'file:///path/to/output.mp4',
  width:     1080,
  height:    1920,
  frameRate: 30,
  quality:   'high',  // 'low' | 'medium' | 'high', or set videoBitrate explicitly

  tracks: [
    {
      type: 'video',
      clips: [
        {
          uri:              'file:///clip-a.mp4',
          startTime:        0,
          duration:         5,
          filter:           'warm',
          filterIntensity:  0.6,
        },
        {
          uri:                'file:///clip-b.mp4',
          startTime:          4,   // overlap by 1 s for the transition
          duration:           5,
          transition:         'crossfade',
          transitionDuration: 1,
        },
      ],
    },
    {
      type: 'audio',
      clips: [
        {
          uri:             'file:///music.mp3',
          startTime:       0,
          duration:        9,
          volume:          0.8,
          fadeOutDuration: 1.5,
        },
      ],
    },
    {
      type: 'text',
      clips: [
        {
          text:      'Hello World',
          startTime: 0.5,
          duration:  3,
          x:         0.5,
          y:         0.15,
          textStyle: {
            fontSize:   52,
            fontWeight: 'bold',
            color:      '#FFFFFF',
            shadowColor:  '#000000',
            shadowRadius: 4,
          },
        },
      ],
    },
  ],
});

Audio & volume

Audio is mixed from all audio and video tracks. Per-clip volume controls:

  • volume — flat multiplier 0–1
  • fadeInDuration / fadeOutDuration — linear ramp at start/end
  • volumeEnvelope.keyframes — time-based automation (overrides fade when provided)
// Volume automation example
volumeEnvelope: {
  keyframes: [
    { time: 0,   volume: 0   },  // start silent
    { time: 1,   volume: 1   },  // ramp up over 1 second
    { time: 8,   volume: 1   },  // hold
    { time: 9.5, volume: 0   },  // ramp out
  ],
}

MediaEnginePreview

A native Expo view. Renders video and audio tracks at ~30 fps using the export-accurate pipeline. Import as a named export.

import { MediaEnginePreview } from '@projectyoked/expo-media-engine';

useCompositionOverlays

Returns the text and image clips active at currentTime, with all transforms and keyframe animations resolved. Memoized — only recalculates when config or currentTime changes.

import { useCompositionOverlays } from '@projectyoked/expo-media-engine';

const overlays = useCompositionOverlays(config, currentTime);
// overlays: ActiveOverlay[]

stitchVideos

Concatenate videos end-to-end in the order provided. Uses a fast passthrough path where possible; falls back to full transcoding if sources are incompatible.

await MediaEngine.stitchVideos(
  ['file:///clip1.mp4', 'file:///clip2.mp4', 'file:///clip3.mp4'],
  'file:///output.mp4'
);

compressVideo

Re-encode a video at a lower bitrate or resolution. Useful for reducing file size before upload.

await MediaEngine.compressVideo({
  inputUri:  'file:///input.mp4',
  outputUri: 'file:///compressed.mp4',
  quality:   'medium',   // 'low' (1 Mbps) | 'medium' (4 Mbps) | 'high' (8 Mbps)
  // Or set explicitly:
  // bitrate: 3_000_000,
  // maxWidth: 1280,
  // maxHeight: 720,
  codec: 'h265',         // Android: H.265 (~40% smaller than H.264 at same quality)
});
FieldTypeDescription
inputUristringSource file. Required.
outputUristringDestination file. Required.
qualitystring'low' | 'medium' | 'high'. Ignored if bitrate is set.
bitratenumberExplicit video bitrate in bps.
audioBitratenumberAudio bitrate in bps. Default: 128000.
width / heightnumberExplicit output dimensions.
maxWidth / maxHeightnumberConstrain dimensions proportionally.
frameRatenumberOutput frame rate.
codecstring'h264' | 'h265' (Android only). Default: 'h264'.

Utilities

getWaveform

Returns a normalized array of RMS amplitude values (0–1) for use in timeline waveform visualizations.

const amplitudes = await MediaEngine.getWaveform('file:///audio.mp3', 200);
// number[] of length 200

extractAudio

Pulls the audio track from a video file and saves it as .m4a.

await MediaEngine.extractAudio('file:///video.mp4', 'file:///audio.m4a');

isAvailable

Returns true if the native module is linked. Check before calling any other function if you need a graceful fallback (e.g. in Storybook or web).

if (!MediaEngine.isAvailable()) {
  console.warn('Native module not linked — rebuild the dev client.');
}
